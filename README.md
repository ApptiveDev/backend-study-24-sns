# backend-study-24-sns (앱티브 백엔드 24기 스터디 과제)
## 1주차 과제
- SNS에 필요한 게시글, 사용자 엔티티 생성 및 관계 매핑
- 게시글 관련 CRUD API 만들어오기 (게시글 작성, 조회, 수정, 삭제)
- 선택사항) Spring Data JPA을 이용하기 전 JdbcClient를 통해 개발해보세요!

## 2주차 과제
- 좋아요 및 댓글 엔티티 생성
- 댓글 CRUD 만들어오기

## 3주차 과제
- JWT 토큰을 이용해서 인증을 구현 
- filter 방식 보다는 Intercepter를 이용해서 인증을 구현하는 것을 추천
- Intercepter에서 prehandle을 잘 작성해 보시면서 인증 흐름에 대해서 설계해보고, JWT payload에 어떤 값을 넣어서 인증을 처리할 지에 대해서 잘 고민해 보세요! 
- Access Token 먼저 구현, Refresh Token을 이용해서 Access Token 재발급 로직 구현 
- Refresh Token은 JWT 형식이 아닌 단순 문자열로 만들어서 DB에 넣어두고. 재발급 시에 DB에 값을 확인해서 처리하도록 만드시면 됩니다!

## 4주차 과제
- 팔로워 팔로잉 기능을 구현
- html을 기반으로 하여 백엔드 데이터를 동적 화면에 반영하는 도구인 thymeleaf로 로그인 화면을 만들고, 해당 화면에서 로그인을 처리한 후 인증 성공 시 화면 구현
(참고로, thymeleaf를 이용해 화면을 렌더링하는 것은 AI를 활용하여 코드를 생성하셔도 됩니다. )

## 5주차 과제
- AWS 의 EC2를 이용해서 docker를 사용하지 않고 1-100까지 모든 배포 과정을 체험
- JDK도 설치하고, git clone을 통해서 소스 코드를 EC2 내부에 올리셔도 좋고, SCP 명령어를 통해서 .jar 파일 자체를 로컬 to EC2로 올리셔도 괜찮습니다. 
- 주의할 점은 jar파일을 실행하실 때 nohup 같이 백그라운드에서 계속 실행되는 명령어를 통해서 올리셔야 저희가 해당 IP로 접근해서 확인이 가능합니다! 
- docker를 이용한 배포는 수동 배포를 진행하고 난 후 시도해보세요! 얼마나 편리한지를 몸으로 직접 느끼게 될 것입니다!
